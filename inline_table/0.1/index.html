Python inline_table Module API Reference¶

inline-table - Embedded text tables in Python code.

inline_table is a Python module for embedding text tables into source-code. We can write source-code just like a design document.

inline_table.compile(text, **variables)[source]¶

Compile a table text to a Table object.

Parameters:
  • text (string) – a table text
  • variables (dict) – values passed to the table
Returns:

a table object
Return type:

Table
Raises:

TableMarkupError – the text format is incorrect
Example:
>>> text = '''
... ============ ======== ==========
...  age (cond)   gender  call (str)
... ============ ======== ==========
...       a < 0    N/A
...  0 <= a < 2     *     baby
...  0 <= a < 7     *     kid
...  7 <= a < 18    M     boy
...  7 <= a < 16    F     girl
... 18 <= a         M     gentleman
... 16 <= a         F     lady
...       *         *     man
... ============ ======== ==========
... '''
>>> table = compile(text, M='male', F='female')
>>> table.select(age=24, gender='female')
Tuple(age=24, gender='female', call='lady')

Table Text Formats:

inline_table supports the following format as a table text:

  • reStructuredText Simple Tables,
  • reStructuredText Grid Tables,
  • Markdown Tables.

The format of the table text is estimated automatically.

Passing Values:

We can pass values to the table with the variables keyword arguments. Values can be passed with the following syntax:

compile(text,
        <name>=<value>, <name>=<value>, ...)

Here <name> are variable names written in the table text.

Special Values:

Two special values can be set to the table.

Name Directive Description
Wild Card * Matches any value. The universal set.
Not-Applicable N/A N/A rows are never returned for queries.

If a string in a cell of the table text equals a directive, the cell is evaluated as the special value.

Note that the string column type does not support the special values.

Column Types:

We can specify a column type with adding a directive to the header row. The difference among column types is how the strings in each cell are evaluated. inline_table provides five column types:

Column Type Directive Evaluated As Set?
Value No Directive, (value), (val) Python literal no
String (string), (str) String. Not support * and N/A. no
Condition (condition), (cond) Conditional statement. Use the 1st letter of the label yes
Regex (regex), (re) Regular expression yes
Collection (collection), (coll) Collection of values yes

A cell in a set column type represents multiple values. The upper table and the lower table in the following act almost similarly (but not strictively equal).

========= ===
a (coll)   b
========= ===
(1, 2, 3)  4
========= ===

======= ===
a (val)  b
======= ===
   1     4
   2     4
   3     4
======= ===

select(**condition)[source]¶

Get the first row that matches the condition.

Parameters:

condition – pairs of a column label and its value
Returns:

the fist matched row
Return type:

Tuple (named tuple)
Raises:

LookupError – no applicable row is found for the condition
Example:
>>> t = compile('''
... === =====
... key value

Get all rows that match the condition.

condition – pairs of a column label and its value

list of matched rows

list of named tuples

LookupError – a key in the condition is invalid

... key value

contains(values)[source]¶

Check if this table contains given values.

In-statement is also available.

A row contain N/A returns False.

Example:
>>> t = compile('''
... === ===
...  A   B
... === ===
...  1   2
... === ===''')
>>> {'A': 1} in t
True
>>> (1, 2) in t
True
iterator()[source]¶

Return an iterator object.

The iter build-in function and for-loops are also available.

A row that contains the not-applicable value is skipped.

Example:
>>> t = compile('''
... === ===
...  A   B
... === ===
...  1   2
...  2  N/A
...  3   6
...  *   0
... === ===''')
>>> i = iter(t)
>>> next(i)
Tuple(A=1, B=2)
>>> next(i)

union(other)[source]¶

Concatenate two tables.

Tables can be concatenated also with + operator.

Two tables must have the same width, the same labels and the same type columns.

Parameters:other – a table to be concatenated
Returns:the concatenated table
Raises:TypeError – width, labels or columns types are different
join(other)[source]¶

Join two tables.

Tables can be joned also with * operator.

This method behaves like NATURAL INNER JOIN in SQL.

Parameters:

other – a table to be join
Returns:

the joined table
Example:
>>> t1 = compile('''
... | A | B |
... |---|---|
... | 1 | 1 |
... | 2 | 2 |''')
>>> t2 = compile('''
... | A (cond)   | C |
... |------------|---|
... | A % 2 == 0 | 0 |
... | A % 2 == 1 | 1 |''')
>>> t3 = t1.join(t2)  # ``t1 * t2`` is equivalent
>>> t3.select_all()
[Tuple(A=1, B=1, C=1), Tuple(A=2, B=2, C=0)]
